# games-store

## 📘 Описание проекта
**games-store** — это полнофункциональное веб-приложение для управления магазином видеоигр, разработанное на платформе **Node.js**.  
Система предоставляет инструменты для администрирования каталога игр, учета клиентов и продавцов, а также фиксации транзакций покупок. Проект демонстрирует архитектуру MVC, работу с реляционной базой данных PostgreSQL и реализацию безопасной системы аутентификации.

---

## 🧩 Основные возможности
- **Управление каталогом игр:** Полный CRUD (создание, чтение, обновление, удаление) для игр с поддержкой жанров, разработчиков и обложек.
- **CRM-система:** Ведение баз данных клиентов и продавцов с валидацией контактных данных.
- **Учет продаж:** Регистрация покупок с связью между клиентом, игрой, платформой и конкретным продавцом.
- **Безопасность:** 
  - Регистрация и авторизация пользователей с хешированием паролей (bcrypt).
  - Защита маршрутов с помощью JWT-токенов.
  - Изоляция сессий и защита от CSRF.
- **Интерфейс:** Динамическая генерация страниц на шаблонизаторе Pug с использованием Flash-сообщений для обратной связи.

---

## ⚙️ Используемые технологии
- **Runtime:** Node.js
- **Фреймворк:** Express.js
- **База данных:** PostgreSQL
- **ORM/Driver:** node-postgres (pg)
- **Шаблонизатор:** Pug
- **Безопасность:** 
  - `jsonwebtoken` (JWT) для аутентификации
  - `bcrypt` для хеширования паролей
  - `helmet`, `cors` (опционально)
- **Утилиты:** 
  - `dotenv` для управления переменными окружения
  - `express-session` и `connect-flash` для работы с сессиями
  - `cookie-parser` для обработки куки

## ▶️ Запуск проекта

### 1. Требования
Перед запуском убедитесь, что установлены:
- **Node.js** (версия 14+)
- **PostgreSQL** (работающий сервер БД)

### 2. Установка зависимостей
Склонируйте репозиторий и установите необходимые пакеты:
```bash
npm install
```

### 3. Настройка базы данных и окружения
1. Создайте базу данных в PostgreSQL (например, `games_veb`) и необходимые таблицы.
2. Создайте файл `.env` в корне проекта на основе шаблона:
   ```bash
   cp .env.example .env
   ```
3. Отредактируйте `.env`, указав актуальные данные для подключения:
   ```env
   # База данных
   DB_USER=postgres
   DB_PASSWORD=ваш_пароль_от_бд
   DB_HOST=localhost
   DB_PORT=5432
   DB_NAME=games_veb

   # Безопасность
   JWT_SECRET=ваш_секретный_ключ_jwt
   JWT_EXPIRES_IN=1h
   SESSION_SECRET=ваш_секретный_ключ_сессии

   # Сервер
   PORT=3000
   NODE_ENV=development
   ```

### 4. Настройка аутентификации (Важно!)

Для включения полноценной системы аутентификации выполните следующие шаги в файле `app.js`:

#### 4.1 Включить проверку JWT
Раскомментируйте строку с middleware аутентификации:
```javascript
app.use(authenticateJWT);
```

#### 4.2 Удалить тестового пользователя
Закомментируйте или удалите middleware, который добавляет тестового пользователя:
```javascript
// app.use((req, res, next) => {
//   req.user = { userid: 1, username: "test_user" };
//   next();
// });
```

### 5. Запуск приложения
Запустите сервер командой:
```bash
node app.js
```
Откройте браузер и перейдите по адресу: [http://localhost:3000](http://localhost:3000)

## 🧠 Назначение
Проект предназначен для:
- Демонстрации навыков разработки серверной части на Node.js.
- Изучения принципов работы с реляционными базами данных и SQL-запросами.
- Реализации безопасных механизмов аутентификации и авторизации в веб-приложениях.
- Использования в качестве шаблона для создания систем учета товаров и услуг.
